# Market Intel — プロジェクトビジョン

## 一言で言うと

**アプリ制作者・インフルエンサーが「自分の立ち位置」を知り、「次の一手」を見つけるためのインテリジェンスツール。**

---

## 誰のためのツールか

| ペルソナ | 具体例 | 抱えている課題 |
|----------|--------|---------------|
| **アプリ制作者** | 個人〜小規模チームでモバイルアプリを運営している開発者 | 競合の動向やユーザーの反応を追う余裕がない。感覚で意思決定している |
| **インフルエンサー / コンテンツクリエイター** | YouTube・SNSで発信し、自分のブランドを育てている個人 | 市場のトレンドや自分のポジションを客観的に把握できない |

共通点: **自分のプロダクトやブランドを持っているが、周辺環境を体系的に監視するリソースがない。**

---

## 何を解決するのか — 3つのコア機能

### 1. 現在地の把握 — 「今どこにいるのか」

対象（自分のアプリ・チャンネル）と周辺環境を一画面で整理する。

- **マクロ環境**: 市場全体のトレンド、業界ニュース、季節性
- **競合の動向**: ライバルアプリの順位・レビュー・広告出稿、競合チャンネルの動き
- **ユーザーの声**: 自分のアプリのレビュー、SNSでの評判、センチメント

> 「自分は今、追い風の中にいるのか、逆風の中にいるのか」が一目でわかる。

### 2. 推移の確認 — 「過去からどう変わってきたか」

時系列でデータを追い、変動の因果関係を紐付けて振り返れるようにする。

- トレンドの上昇・下降がいつ起きたか
- その前後に何が起きていたか（競合のアプデ、自分のイベント、市場の変化）
- どのアクションがどの結果に繋がったか（因果ログの蓄積）

> 「あの施策が効いたのか、市場の追い風だったのか」を区別できる。

### 3. 次の一手の予測 — 「何をすればより良くなるか」

蓄積された因果パターンから、成功確率の高いアクションを提示する。

- 過去に似た状況で何が効いたか
- 信頼度の高いパターン（繰り返し検証されたもの）を優先表示
- 逆に、やっても効果がなかったパターンを自動淘汰

> 「勘」ではなく「蓄積された根拠」に基づく意思決定を支援する。

---

## なぜこの3つなのか

```
現在地がわからない → 判断の前提がない
推移がわからない   → 何が効いたか学べない
予測がない         → 毎回ゼロから考える

この3つが揃って初めて「インテリジェンス」になる。
データ収集だけでは情報、分析だけでは知識。
行動に繋がる示唆まで届けて初めて知恵（Intelligence）。
```

## 既存資産との接続

現在の4層モデルはそのまま活用できる。フレーミングを変えるだけ。

| 4層モデル | 旧フレーム（ドメイン汎用） | 新フレーム（クリエイター向け） |
|-----------|---------------------------|-------------------------------|
| **L1 マクロ** | 市場全体の潮流 | 自分がいる市場の追い風・逆風 |
| **L2 競合** | 直接比較可能な対象 | ライバルの動き（何をして、どう変わったか） |
| **L3 ユーザー** | 末端の反応と感情 | 自分のユーザー/フォロワーの声と行動 |
| **L4 因果** | 因果パターンの蓄積 | 「こうしたらこうなった」の学習エンジン |

### 変わるもの・変わらないもの

**変わらない**:
- 4層収集モデル
- 異常値検出・トレンド分析・因果候補抽出のパイプライン
- 因果ログ自動蓄積・学習エンジン（プロジェクトの中核価値）
- ドメイン設定による対象切り替え

**変わる**:
- ターゲットユーザーが「自分」から「アプリ制作者・インフルエンサー」に
- ダッシュボードの構成が「データ閲覧」から「現在地→推移→次の一手」の3ステップに
- 因果ログの見せ方が「パターン一覧」から「行動提案」に

---

## ドメインの位置づけ — 整理

| ドメイン | 旧位置づけ | 新位置づけ |
|----------|-----------|-----------|
| **ゲーム市場** | 本業武器 | **メインドメイン**: アプリ制作者向けの第一ユースケース |
| **株式** | 個人資産運用 | **将来拡張**: クリエイターの事業環境分析（関連企業の動向）として再定義可能 |
| **競馬** | 副収入 | **削除候補 or 別プロジェクト**: ペルソナと整合しない |

> 「アプリ制作者・インフルエンサー向けツール」と言ったときに、競馬が入っていると説明がつかない。
> 株式は「自分の業界の上場企業の動向を追う」と再解釈すればギリギリ整合するが、優先度は低い。

## ダッシュボードの再構成案

現在の3タブ（アプリ情報 / ユーザー / 業界・イベント）を、3つのコア機能に対応させる：

```
┌─────────────────────────────────────────────────┐
│  Market Intel                                    │
├──────────┬──────────┬──────────┬────────────────┤
│  現在地   │  推移     │  次の一手 │               │
│ Position │ History  │ Actions  │               │
└──────────┴──────────┴──────────┴────────────────┘

[現在地] タブ
  - スコアカード: 自分のアプリの主要KPI（評価、DL数、レビュー傾向）
  - 市場ポジション: 競合との相対位置
  - 環境サマリー: マクロトレンドの追い風/逆風インジケータ

[推移] タブ
  - タイムライン: 主要指標の時系列チャート
  - イベントオーバーレイ: 自分の施策・競合の動き・市場イベントを重畳表示
  - 因果ハイライト: 「この変動はこれが原因の可能性が高い」を注釈表示

[次の一手] タブ
  - 推奨アクション: 信頼度の高い因果パターンに基づく提案
  - リスク警告: 検出された逆風パターン
  - パターン一覧: 蓄積された因果ログ（フィルタ・信頼度順）
```

---

## 成功の定義

このツールが「うまくいった」と言えるのは：

1. **使い続けられる** — 週1回はダッシュボードを開いて意思決定に使う
2. **蓄積が効く** — 3ヶ月後、6ヶ月後に因果ログの精度が上がっている実感がある
3. **説明できる** — 「自分のアプリの立ち位置と次にやるべきことを教えてくれるツール」と一文で言える
4. **他の人にも使える** — 対象アプリを変えれば、別のクリエイターも同じ価値を得られる
