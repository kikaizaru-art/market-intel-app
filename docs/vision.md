# Market Intel — プロジェクトビジョン

## 一言で言うと

**対象（アプリ・人物・企業など）の情報と周辺環境を整理し、今の立ち位置・過去の推移・次の一手を導くインテリジェンスツール。**

---

## 何を解決するのか — 3つのコア機能

### 1. 現在地の把握 — 「今どこにいるのか」

対象と周辺環境を一画面で整理する。

- **マクロ環境**: 対象が属する市場や業界全体のトレンド、ニュース、季節性
- **競合の動向**: 比較対象の動き（順位、評判、広告、業績）
- **ユーザー/市場の反応**: レビュー、SNSセンチメント、オッズ、出来高など末端の反応

> 「今、追い風の中にいるのか、逆風の中にいるのか」が一目でわかる。

### 2. 推移の確認 — 「過去からどう変わってきたか」

時系列でデータを追い、変動の因果関係を紐付けて振り返れるようにする。

- いつ、何が起きて、何が変わったか
- そのアクションが本当に効いたのか、外部要因だったのか
- 因果ログとして蓄積し、後から振り返れる

> 「あの施策が効いたのか、市場の追い風だったのか」を区別できる。

### 3. 次の一手の予測 — 「何をすればより良くなるか」

蓄積された因果パターンから、成功確率の高いアクションを提示する。

- 過去に似た状況で何が効いたか
- 信頼度の高いパターン（繰り返し検証されたもの）を優先表示
- やっても効果がなかったパターンを自動淘汰

> 「勘」ではなく「蓄積された根拠」に基づく意思決定を支援する。

---

## なぜこの3つなのか

```
現在地がわからない → 判断の前提がない
推移がわからない   → 何が効いたか学べない
予測がない         → 毎回ゼロから考える

この3つが揃って初めて「インテリジェンス」になる。
データ収集だけでは情報、分析だけでは知識。
行動に繋がる示唆まで届けて初めて知恵（Intelligence）。
```

## 4層モデル — なぜ汎用に効くのか

このツールの方法論の核は **4層収集モデル** にある。
対象が何であっても、周辺環境は同じ4層で構造化できる。

| 4層モデル | 意味 | アプリ | 株式 | 競馬 |
|-----------|------|--------|------|------|
| **L1 マクロ** | 対象を取り巻く大きな潮流 | 市場トレンド、業界ニュース | 金利・為替・セクター指数 | JRA売上、馬場状態、天候 |
| **L2 競合** | 比較対象の動き | 競合アプリの順位・レビュー・広告 | 同業他社の決算・株価 | 各馬の成績・調教・騎手 |
| **L3 ユーザー** | 末端の反応（先行指標） | アプリレビュー、SNSセンチメント | SNS・掲示板の話題量 | オッズ推移、SNS予想の偏り |
| **L4 因果** | 蓄積された「こうしたらこうなった」 | イベント→トレンド異常値の相関 | 決算発表→翌日値動き | 馬場×血統×距離→回収率 |

**対象を差し替えるだけで、同じ収集・分析・学習の構造が動く。**

## ドメインの位置づけ

| ドメイン | 役割 | 状態 |
|----------|------|------|
| **ゲーム市場** | 最初のユースケース。実データ収集済みで最も成熟 | 実装済み |
| **株式** | 個人の資産運用。マクロ指標 + 企業決算 + SNSの構造 | 設計済み |
| **競馬** | 副収入。条件×成績の因果パターンが最も検証しやすい | 設計済み |

優先順位はあるが、どれも4層モデルの正当なユースケース。
「このドメインだから使えない」という制約は設けない。

## ダッシュボードの構成方針

3つのコア機能に対応するタブ構成：

```
┌─────────────────────────────────────────────────┐
│  Market Intel                                    │
├──────────┬──────────┬──────────┬────────────────┤
│  現在地   │  推移     │  次の一手 │               │
│ Position │ History  │ Actions  │               │
└──────────┴──────────┴──────────┴────────────────┘

[現在地] タブ
  - スコアカード: 対象の主要KPI
  - ポジション: 競合・比較対象との相対位置
  - 環境サマリー: マクロトレンドの追い風/逆風インジケータ

[推移] タブ
  - タイムライン: 主要指標の時系列チャート
  - イベントオーバーレイ: アクション・競合の動き・市場イベントを重畳表示
  - 因果ハイライト: 「この変動はこれが原因の可能性が高い」を注釈表示

[次の一手] タブ
  - 推奨アクション: 信頼度の高い因果パターンに基づく提案
  - リスク警告: 検出された逆風パターン
  - パターン一覧: 蓄積された因果ログ（フィルタ・信頼度順）
```

## 成功の定義

このツールが「うまくいった」と言えるのは：

1. **使い続けられる** — 定期的にダッシュボードを開いて意思決定に使う
2. **蓄積が効く** — 時間とともに因果ログの精度が上がっている実感がある
3. **説明できる** — 「対象の立ち位置と次にやるべきことを教えてくれるツール」と一文で言える
4. **対象を変えても動く** — ドメイン設定を差し替えれば、別の対象でも同じ価値を得られる
