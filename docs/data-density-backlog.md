# データ情報量の改善バックログ

「使う気にならない」の主因は **データの時間軸が無い × 情報密度が低い × 自分の施策データが無い** こと。
UI 改善と並行して、収集データ自体の質と量を上げる施策を優先順に潰していく。

診断の根拠は `docs/vision.md` と本ファイル末尾の「現状診断」を参照。

---

## 優先順リスト

### 即効性・高 (1-2日)

- [x] **1. レビュー本文の保存** ★最初に着手
  - 現状: `data/history/{domain}.json` に score/ratings/histogram/version しか残らない。レビュー本文は `collected.json` のスナップショットに30件あるだけで時系列で消えていく
  - 打ち手: history に `reviewTexts[appId]` を追加し、レビューIDで重複除去しながら最大 N 件を rolling 保存。バージョン・日付・スコアを紐付ける
  - 効果: 「何が起きたか」の情報密度が10倍。L3 (ユーザー) の本丸が機能し始める
  - 依存: なし

- [x] **2. `trends` 履歴蓄積バグの調査と修正**
  - 現状: `data/history/memento-mori.json` の `trends` が **0点**。設計値は週次52週
  - 打ち手: `src/collectors/index.js` の `mergeWithHistory` の trends 分岐を実測。`results.trends.weekly` が空配列で来ている疑い → コレクター側 (`trends.js` / `fetch_trends.py`) の出力を確認
  - 効果: 週次52週がすぐ埋まり始める。「推移」タブが本来の姿になる
  - 依存: なし

### 中期 (1週間)

- [ ] **3. Twitter/X 検索の追加**
  - 現状: 日本産ゲームの議論は X と 5ch と YouTube にあるが、収集対象外。Reddit は 1日2件で実質ゼロ
  - 打ち手: 公式 API は高いので nitter インスタンス or snscrape 等の代替経路を検証。レート制限と利用規約に注意
  - 効果: ユーザーの声の量と即時性が桁違いに上がる
  - 依存: なし (ただしソース選定で詰まる可能性あり)

- [x] **4. YouTube Data API で配信数・再生数の蓄積**
  - 現状: ゲームの先行指標として配信者の動きが効くが未収集
  - 打ち手: YouTube Data API v3 (`channels.list` + `playlistItems.list` + `videos.list`) で対象チャンネルの登録者数・総再生数・直近30日の投稿数と平均再生数を取得し、週次26週で履歴蓄積
  - 効果: 配信者系ゲームの伸び/減衰の先行指標が手に入る + インフルエンサードメインの L2 が実データで動く
  - 依存: `YOUTUBE_API_KEY` 環境変数 (無料枠1日10,000 quota, チャンネル毎3 quota)
  - 実装: `src/collectors/youtube-channels.js` を実API対応 + `index.js` に6番目コレクターとして統合 (対象に `youtube_channel_id` がある場合のみ実行、履歴は `history.youtubeChannels[id][]`)

- [x] **5. ニュースを app 単位でタグ付け**
  - 現状: 4Gamer/AUTOMATON/Inside/GameSpark の全ゲーム RSS が 40件ベタッと出ているだけ。L1 マクロと L2 競合が分離されていない
  - 打ち手: ニュース本文/タイトルに各 target の name (および別名) が含まれているか判定し、`appTags: [...]` を付与。ダッシュボードで app 別フィルタ可能に
  - 効果: 「メメントモリ関連ニュース」「競合関連ニュース」が分かる
  - 依存: なし

### 高価値・仕込み系

- [x] **6. アプデ履歴の自動イベント化**
  - 現状: `recentChanges` (What's New) は取れているが、バージョン更新を「イベント」として因果エンジンに流していない
  - 打ち手: 月次 review history の version 変化を検出 → 自動的にイベントログに追加 → `autoMemo.js` の `validateAnomalyEvent` の入力に使う
  - 効果: 「自分の施策」側のデータが因果学習の片翼として動き始める
  - 依存: 1番 (本文保存) があると更に強い

- [x] **7. イベントカレンダー入力の軽量UI**
  - 現状: vision.md:251-279 のハイブリッド入力 (ワンタップ + メモ + 自由記帳) が未実装
  - 打ち手: 自動検出メモに `userContext` フィールドを追加。日付ピッカー + 4選択肢 + メモ入力の最小UI
  - 効果: ガチャ・コラボ・広告などの「外から見えない自分の施策」を1タップで記録 → 因果ログの精度が劇的に上がる
  - 依存: なし。ただし 1, 2 の方が即効性高い
  - 実装: `EventQuickInput.jsx` — ドメイン別プリセット (ワンタップ選択 + 日付 + メモ + 自由記帳) で因果関係パネルに統合

## 現状診断 (2026-04-15 時点)

`data/history/memento-mori.json` の実測:

| 層 | 設計値 | 現状 | ギャップ |
|---|---|---|---|
| reviews (月次) | 12ヶ月 | 1ヶ月 | -92% |
| trends (週次) | 52週 | 0点 | 完全欠落 |
| ranking (日次) | 90日 | 1日 | -99% |
| community (日次) | 30日 | 1日 | -97% |
| news | 履歴蓄積なし | 40件 (スナップショット) | 時系列情報ゼロ |

質の問題:
- Reddit の `gachagaming` で 1日2件 → ユーザーの声を拾えていない (3番で対応)
- ニュースが app 別に分かれていない (5番で対応)
- レビュー本文が捨てられている (1番で対応)
- 自分の施策データが無い (6, 7番で対応)
